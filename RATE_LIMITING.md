# Rate Limiting

Система ограничения частоты запросов (rate limiting) защищает приложение от спама и злоупотреблений.

## Реализованные ограничения

| Действие | Лимит | Период |
|----------|-------|--------|
| Создание постов | 10 | 5 минут |
| Создание комментариев | 20 | 5 минут |
| Лайки | 60 | 1 минута |
| Отправка сообщений | 30 | 5 минут |
| Запросы в друзья | 20 | 1 час |
| Репорты | 5 | 1 час |
| Создание событий | 5 | 1 час |
| Записи в календарь | 10 | 1 час |
| Загрузка фото | 20 | 1 час |

## Как это работает

Rate limiting реализован в `src/lib/rate-limit.ts` с использованием in-memory хранилища. Для каждого пользователя и действия отслеживается:

- Количество запросов в текущем временном окне
- Время сброса счетчика

Когда пользователь превышает лимит, он получает сообщение с указанием времени ожидания.

## Защищенные действия

### Посты и взаимодействия
- `src/app/actions/post.ts` - создание постов
- `src/app/actions/interactions.ts` - лайки и комментарии
- `src/app/actions/comments.ts` - создание комментариев

### Социальные функции
- `src/app/actions/friends.js` - запросы в друзья
- `src/app/actions/photo.js` - загрузка фотографий

### Контент и модерация
- `src/app/actions/report.ts` - репорты контента
- `src/app/actions/events.ts` - создание событий
- `src/app/actions/calendar.ts` - записи в календарь

## Сообщения об ошибках

Пользователи видят понятные сообщения:
- "Rate limit exceeded. Please try again in 2m 30s"
- "Rate limit exceeded. Please try again in 45s"

## Очистка памяти

Старые записи автоматически удаляются каждые 5 минут для предотвращения утечек памяти.

## Будущие улучшения

Для production рекомендуется:
- Использовать Redis вместо in-memory хранилища
- Добавить IP-based rate limiting
- Реализовать разные лимиты для верифицированных пользователей
- Добавить мониторинг и алерты при превышении лимитов
